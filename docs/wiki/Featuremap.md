## Feature Map

```mermaid

flowchart LR
    root((Lost & Found QR Platform)) 
        root --> T(Tag Creation/Management)
            T --> T1(QR Code Generator)
                T1 --> T11[online in Lambda]
                T1 --> T12[offline in JS]
            T --> T2(Multi-item tags)
            T2 --> T21(Same ID for different items)
              
        root --> OS(Ownership)
            OS --> OS1(Tag initial anonymous)
                OS1 --> OS11(generation as unique IDs)
                OS1 --> OS12(unclaimed)
                OS1 --> OS13(allows batch generation)
            OS --> OS2(Claim Tag)
                OS2 --> OS21(connect Tag ID with UserID)
                OS2 --> OS22(Scan of unclaimed Tags provides optional Ownership)
            OS --> OS3(Multi Owner)
                OS3 --> OS31(share tags)
                OS3 --> OS32(invite additional owner by link)
            OS --> OS4(Unclaim Tag)
                OS4 --> OS41(delete Tag from Database)
                OS4 --> OS42(give up Ownership)
                OS4 --> OS43(allows re-use of QR codes and physical tags)
        root --> F(Finder flow)
            F --> F1(QR scan without app)
            F --> F2(Send message)
                F2 --> F21(anonym)
                F2 --> F22(relay)
            F --> F3(Attachments)
                F3 --> F31(Picture of item)
                F3 --> F32(Location)
                    F32 --> F321(very corse for verification, e.g. 2km)
                    F32 --> F322(fine for handover process)
            F --> F4(Anonym)
                F4 --> F41(no login process)
                F4 --> F42(session based, not user)
        root --> M(Messaging)
            M --> M1(Session token with seed)
            M --> M2(Inbox buffering)
            M --> M3(rate limitation)
            M --> M4(no contact data exchange by platform)
            M --> M5(Filter to mask contact data)
            M5 --> M51(Disable if both sides agree)
        root --> P(Privacy / Data Handling)
            P --> P1(Pseudonymous user ID only)
            P --> P2(Automatic deletion via TTL)
            P --> P3(Transparent privacy page)
            P --> P4(No cross-tag tracking/profile)
        root --> U(UX & Platform)
            U -->U1(Browser only, PWA ready)
            U -->U2(QR scan & labeling mobile & desktop)
            U -->U3(Native sharing via Web Share API, no email usage on platform)
            U -->U4(No app required)
        root --> A(API & Architecture)
            A --> A1(AWS API Gateway + Lambda e.g public/admin)
            A --> A2(DynamoDB)
            A2 -->A21(minimal data only)
            A --> A3(Auth via Cognito OIDC + hash of ID with seed)
        root --> OP(Optional)
            OP --> OP1(Admin interface)
            OP1 --> OP11(logs)
            OP1 --> OP12(TTL)
 
````
