> **Status**: Planned – basis for refactor sprint

## Overview

The v2 architecture eliminates the code-generation pipeline in favour of
[AWS Lambda Powertools for Python](https://docs.powertools.aws.dev/lambda/python/latest/)
as the single framework for routing, validation, observability and OpenAPI export.

All infrastructure is defined with **AWS CDK (Python)**. There are no static AWS keys
anywhere – GitHub Actions authenticates via **OIDC**.

---

## Stack Layout (3 CDK Stacks)

```
LostAndFoundDatabaseStack
    Owns all DynamoDB tables. RemovalPolicy.RETAIN on all.
    ├── owner_table          PK: owner_hash
    ├── tag_table            PK: tag_id
    ├── owner_session_table  PK: session_token  TTL: expires_at
    └── finder_session_table PK: session_token  TTL: expires_at

LostAndFoundApiStack
    Owns API Gateway REST API + Lambda functions + Shared Layer.
    Receives DatabaseStack as dependency via constructor.
    ├── RestApi (API Gateway)  – single gateway, eu-central-1
    ├── SharedLayer            – pydantic, powertools, cryptography, srptools
    ├── owner Lambda           → /v1/owner/{proxy+}
    ├── finder Lambda          → /v1/finder/{proxy+}
    └── tag Lambda             → /v1/tag/{proxy+}

LostAndFoundUIStack
    Owns frontend hosting.
    ├── S3 Bucket   – private, BLOCK_ALL public access
    └── CloudFront  – OAC (Origin Access Control), free tier
```

### Why 3 stacks and not more?

- `DatabaseStack` is isolated so `cdk destroy` on the API stack never touches data.
- `UIStack` can be deployed independently from the backend.
- Separating DB from API gives clean IAM grant boundaries.

## Lambda Pattern (Powertools)

Every Lambda follows the same 3-file pattern:

### `app.py` – Router registration

```python
from aws_lambda_powertools.event_handler import APIGatewayRestResolver
from routes.onboarding import router as onboarding_router
from routes.auth import router as auth_router

app = APIGatewayRestResolver(enable_validation=True)
app.include_router(onboarding_router, prefix="/v1/owner")
app.include_router(auth_router, prefix="/v1/owner")
```

### `lambda_handler.py` – Entry point

```python
from aws_lambda_powertools.utilities.typing import LambdaContext
from app import app

def handler(event: dict, context: LambdaContext) -> dict:
    return app.resolve(event, context)
```

### `routes/<domain>.py` – Route definition

```python
from aws_lambda_powertools.event_handler import Router
from shared.api.owner.api_owner_model import OnboardingRequest, OnboardingInitResponse
from onboarding_logic import onboarding_logic

router = Router()

@router.post("/onboarding")
def owner_onboarding(body: OnboardingRequest) -> OnboardingInitResponse:
    return onboarding_logic(body)
```

Key properties:
- `enable_validation=True` activates Pydantic request/response validation automatically.
- `Router` objects are imported in `app.py` – this keeps routes testable in isolation.
- Business logic stays in separate `*_logic.py` files (preserved from v1).

## API Gateway Routing

API Gateway performs **prefix-level routing only**. Fine-grained routing is owned by
Powertools inside each Lambda:

```
HTTP Request
    │
    ▼
API Gateway REST API  (single gateway, stage = dev | prod)
    ├── /v1/owner/{proxy+}   ──►  owner Lambda  ──►  Powertools Router
    │                                                ├── POST /v1/owner/onboarding
    │                                                ├── POST /v1/owner/login
    │                                                └── ...
    ├── /v1/finder/{proxy+}  ──►  finder Lambda ──►  Powertools Router
    └── /v1/tag/{proxy+}     ──►  tag Lambda    ──►  Powertools Router
```

CDK definition (per Lambda, in `api_stack.py`):

```python
def _add_proxy(self, root, prefix: str, fn: Function):
    resource = root.add_resource(prefix)
    resource.add_method("ANY", LambdaIntegration(fn))
    resource.add_resource("{proxy+}").add_method("ANY", LambdaIntegration(fn))
```

## OpenAPI Spec Export

The `api/openapi.yaml` is **generated output**, not a maintained input:

```
Code (Pydantic models + Powertools routes)
    │
    ▼  devtools/export_openapi.py
api/openapi.yaml   ←── committed to repo, used for contract tests
    │
    ▼  tests/contract/test_openapi_contract.py
Contract Tests     ←── CI gate on every PR
```

The export script imports all Lambda `app` instances, merges their OpenAPI schemas,
and writes `api/openapi.yaml` + `api/openapi.json`.

## Testing Strategy

| Layer | Tool | Trigger | AWS needed |
|---|---|---|---|
| Unit (runtime) | pytest + moto | every push | ❌ |
| CDK Assertions | aws-cdk-lib.assertions | every PR | ❌ |
| Contract (OpenAPI) | pytest + schemathesis | every PR | ❌ |
| Integration | pytest + httpx | after deploy-dev | ✅ dev-Stage |

### Unit test pattern (Handler-level)

```python
# No HTTP, no network – directly invoke Powertools resolver
from owner.app import app

def test_onboard_success():
    event = {
        "httpMethod": "POST",
        "path": "/v1/owner/onboarding",
        "body": '{"owner_hash": "owner_abc123"}',
        "headers": {}, "queryStringParameters": None,
        "requestContext": {"resourcePath": "/v1/owner/onboarding"}
    }
    result = app.resolve(event, {})
    assert result["statusCode"] == 201
```

## Lambda Layer Build (no Docker required)

All planned crypto libraries ship pre-built `manylinux` wheels on PyPI:

| Library | Usage | Wheel type |
|---|---|---|
| `pydantic` v2 | Models + validation | manylinux (Rust) |
| `aws-lambda-powertools` | Routing, logging, tracing | Pure Python |
| `cryptography` | AES-GCM, HKDF, PBKDF2 | manylinux (OpenSSL) |
| `srptools` | SRP RFC 5054 | Pure Python |
| `mnemonic` | BIP39 Paper Code | Pure Python |

Build command in CI:

```bash
pip install \
  --target dist/layer/python \
  --platform manylinux2014_x86_64 \
  --implementation cp \
  --python-version 3.12 \
  --only-binary=:all: \
  pydantic aws-lambda-powertools cryptography srptools mnemonic
cp -r runtime/shared/src/shared dist/layer/python/
```

## mypy Configuration

Zone-based strictness – strict for own code, pragmatic for third-party:

```toml
[tool.mypy]
python_version = "3.12"
strict = true

[[tool.mypy.overrides]]
module = ["aws_lambda_powertools.*", "boto3.*", "botocore.*", "moto.*"]
ignore_missing_imports = true
disallow_untyped_defs = false
```

Bonus: add `boto3-stubs[dynamodb,s3,kms]` as dev dependency for full DynamoDB type safety.
